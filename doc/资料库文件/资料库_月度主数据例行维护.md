# 资料库：月度主数据例行维护

## 推荐命名

这类工作不要散着说成“价格、BOM、Routing、物料维护”，更适合作为一个固定业务场景来表达。

推荐说法：

- 月度主数据例行维护
- 月度生产与供应链主数据维护
- 月度基础数据校准
- 定期主数据巡检与更新

如果面向 SAP 乙方交付、运维、AMS、制造业客户，优先使用：

**月度生产与供应链主数据维护**

如果面向更泛的企业管理者或公众号读者，优先使用：

**月度主数据例行维护**

## 一句话定义

月度主数据例行维护，是指企业每个月围绕生产、采购、销售、成本、质量等业务环节，对 SAP 中相对稳定但需要持续校准的基础数据进行检查、更新和补全的工作。

这些工作通常不是一次性项目，而是每个月都会重复发生。典型例子包括价格更新、BOM 调整、工艺路线维护、物料参数校准、供应商采购信息维护等。

## 维护项目清单

| 类别 | 维护项目例子 |
| --- | --- |
| 物料主数据 | 物料状态、MRP 类型、采购类型、批量规则、安全库存、计划交货时间、自制生产时间、单位换算、采购/销售/成本视图 |
| BOM 物料清单 | 组件、用量、损耗率、替代料、有效期、BOM 状态、不同工厂/版本的 BOM |
| Routing 工艺路线 | 工序顺序、工作中心、标准工时、准备时间、机器时间、人工时间、外协工序、控制码 |
| 工作中心 | 产能、班次日历、成本中心、作业类型、能力公式 |
| 生产版本 | BOM 与 Routing 的绑定关系、有效期、批量范围、MRP 是否可用 |
| 价格/成本 | 标准价格、移动平均价、计划价格、标准成本估算、采购价格、销售价格、折扣、运费、税等条件记录 |
| 采购主数据 | 采购信息记录、供应商-物料关系、货源清单、配额协议、交期、MOQ、采购单位 |
| 销售主数据 | 客户-物料信息记录、客户专属料号、销售价格、客户折扣、交付规则 |
| 质量/批次相关 | 检验计划、质量状态、批次管理标识、有效期、保质期参数 |
| 数据清理/校验 | 失效价格、过期 BOM、无效 Routing、缺失生产版本、MRP 异常主数据、重复物料、停用物料 |

## 更自然的表达方式

### 偏正式表达

每个月业务团队都会进行一批固定的主数据维护工作，比如价格与成本更新、BOM 调整、工艺路线维护、物料主数据校准，以及采购信息记录、货源清单、生产版本等基础数据的例行检查。

### 偏口语表达

这些不是一次性的项目，而是每个月都要重复做的 SAP 主数据维护，比如价格、BOM、工艺路线、物料参数、供应商采购信息这些。

### 偏 SAP 乙方交付表达

在 SAP 运维和持续优化项目里，很多工作不是故障处理，而是稳定重复发生的基础数据维护。比如每个月的价格更新、BOM 调整、Routing 维护、物料主数据校准、采购信息记录维护、生产版本检查等。这些工作单次看起来不复杂，但长期来看非常消耗顾问和业务关键用户的时间。

## 和 AI 场景的连接点

这个场景适合作为“AI 降低固定作业成本”的例子。

可以强调：

- 这些维护项高度重复，但又不能完全自动化，因为需要业务判断。
- 很多数据维护前，需要先收集 Excel、邮件、工单、历史变更记录和 SAP 当前配置。
- 维护后还要做校验、留痕、生成变更说明、通知相关业务方。
- AI 适合做资料汇总、差异识别、规则校验、说明生成、影响范围初判和记录归档。

可用表达：

每个月固定要做的主数据维护，过去靠顾问人工对 Excel、查 SAP、写说明。AI 不能替你拍板，但可以把资料汇总、差异识别、规则校验、变更说明和记录归档这几步先做起来，让顾问把时间花在判断上，而不是花在重复整理上。

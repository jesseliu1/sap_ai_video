# 措辞口径速查表

> 每篇文章动笔前、定稿前各扫一遍。这张表把母文档 V0.8 里最容易写翻车的地方抽出来，直接当对照清单用。

## 零、定位与分发口径（V0.8 求职重定位）

- **账号本质：** 这是一份「会呼吸的简历」。唯一目的，是让微信好友里能雇你的老板，雇你去帮他们的乙方维护团队搭一套 SAP 维护 AI 系统。号背后没有真实业务、交付团队或咨询业务。
- **买家是能雇你的人**（乙方负责人 / 交付经理 / SAP 圈决策者），不是读者大众；面向一线顾问的内容只是能力背书。
- **分发靠微信温好友天然解决：** 不追爆款、不追完读率、不跟算法较劲、不做搜一搜 SEO，也不以领英 / 脉脉为主渠道。
- **成功的唯一定义：** 好友里能雇你的人，读完更信你。CTA 是声誉，不是转化按钮。
- **调整信号：** 按目标雇主的信任反馈调，不按阅读量 / 传播数据调。
- **不解释 SAP 基础**（读者是内行，一解释就掉价）。

## 二、四根承重柱（V0.8 重排 · 账号底层世界观）

1. **价值在前，边界当调味。** 人设从"我懂 AI 不能做什么"翻成"**我能带来什么、而且拿得出做过的东西**"。边界感不删，但降为佐料，只在解释"方案为什么稳、为什么不坑你"时出场。"防御密度原则"退为背景手法，不再当承重柱——别让过度防御把真正的价值点埋掉。
2. **"入口质量决定 AI 上限"是撞出来的。** 实证锚点：把十年工单喂进 RAG，却发现里面根本没有答案（有效召回极低，连写单本人回看都未必看得懂）。这把"入口 / 数据质量决定 AI 上限"从纸面世界观升级成亲历，是开篇定海针。脏的存量直接喂 AI = "格式漂亮、口气笃定的垃圾"，比没有更危险。
3. **从 L1 切入是策略，不是天花板。** 判断先写哪个，不只看技术多深，更看阻力多低、复现多高、是否适合公开展示。务实从 L1 起步，但内容要让人看见你看得见 L2 / L3 的路——只是知道现在不该急着下水。
4. **人天定价在死、价值定价是机会——这是最强的雇主钩子。** 客户已经不那么为堆人天买单；把省下的工时转到客户肯付溢价的地方、或按成果重新定价，正是乙方老板最焦虑、也最想找人解决的事。经营线往中心放；但自带"不藏价、不瞒客户降本"的反例，守住可信度。

## 三、三层深度的安全说法

- **第一层（不接系统 / 弱关联）：** 低阻力、高复现、最适合公开。但仍要控制客户数据、截图、日志、敏感信息外流。专业度靠"产出密度"体现，不靠"接系统"。从 L1 起步是策略选择，不是天花板——同时让人看见 L2 / L3 的路。
- **轻量第二层（与乙方自身系统关联）：** 仅基于乙方可控的数据导出、离线样本、脱敏工单、测试库、知识库、非生产环境做**辅助分析**。**不写回生产、不自动分派、不自动关单、不自动审批、不自动改客户 SAP。** 适合乙方内部试点。
- **第三层（与客户 SAP 环境关联）：** 价值最高、阻力最高，适合客户授权后的深度项目。
- **硬约束：** 第二、三层每次出现，必须同步说明授权、数据、安全、审计、责任边界，一条都不能省。

## 四、两记雇主钩子文的立场口径（V0.8 经营线往中心放）

**第 04 篇（经营算账 · 雇主钩子①）**
- 标题克制，正文锋利。
- 核心论点：人天模式卖的是工时；客户越来越不愿为整理文档、补证据、写月报这类非专家工时付费——这部分本就在被压价。AI 把正在贬值的非专家工时，换成可重新定价的东西（更快响应、更稳上线、更深根因，或方法论产品化按成果收费）。
- 这是面向背 P&L 的老板的正面算账，往中心放；但**自带"不藏价、不瞒客户降本"的反例守可信度**——不写成教乙方糊弄客户。
- 可保留的正文钩子（不上标题）："AI 压的不是你的毛利，是你账单里那部分客户早就不想付的工时。真正的风险，是对手先用它把人天结构升级了，而你还在靠堆人头守毛利。"

**第 05 篇（客户不让工单进 AI · 雇主钩子② · 立场不可违反）**
- 不得把客户写成"不懂 AI / 过度保守 / 自相矛盾"。客户不是不讲理，而是 AI 的数据边界没被翻译成他们熟悉的审批语言。
- 脊梁三段，缺一不可：
  1. **真红线，绝不硬来**——数据出境、个人信息保护、客户既定安全策略、新增数据处理者、日志留存、供应商与输出责任、员工 Shadow AI。这一段是可信度的锚，证明作者懂分寸。
  2. **很多"不让用"是惯性墙**——怕担责、没人提过、一刀切、"我们一直这么做"。典型如："消费版 ChatGPT 会拿对话去训练"这条顾虑是真的、该尊重（归到上面真红线那半）；但企业版 AI 默认不拿你的数据训练，却仍被连坐一刀切禁掉——这后半截才是惯性墙。区分清楚，别把真顾虑当惯性来调侃，那会削掉你"懂分寸"的可信度。
  3. **作者的本事 = 把假红线翻译成可谈条款，设计客户敢点头的可控试点**——脱敏样本、模拟工单、数据不出系统、人审闸门、小范围灰度。
- 文章任务：把"笼统的不敢给 AI 数据"拆成可判断、可脱敏、可配置、可审批、可试点的清单——并让老板看见：这堵墙，作者能帮他谈下来。

## 七、AI 能做 / 不能做（统一边界口径 · V0.8 已降为调味）

- **能做：** 工单补料、内容摘要、分类建议、相似工单检索、草稿生成、证据整理、缺口检查、数据统计。
- **不能单独决定：** 自动修系统、自动上线、自动审批、自动判断根因、自动替业务验收、自动替人承担责任。
- 一句话：AI 负责信息整理和候选项，最终判断与承诺永远在人（顾问 / 负责人 / 客户关键用户 / 审批角色）这里。
- **证据场重心（V0.8）：** 边界不再当承重柱、只作调味；呈现重心前移到"做成了什么"——脏输入 → AI 处理前的问题 → 结构化交付物 → 审核通过 → 可复用模板 / 检查清单，让交付物 / 作品当主角。

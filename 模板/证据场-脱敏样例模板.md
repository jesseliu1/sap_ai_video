# 证据场·脱敏样例模板（六要素）

> 来源：母文档 V0.8 · 证据场重心（§15.5 三线权重）。证据场是把"我懂 AI 不能做什么"翻成"我能带来什么、而且拿得出做过的东西"的主力——**交付物 / 作品当主角，边界只当调味**。
> 「必配证据场」的篇目优先级最高；每篇尽量给至少一个贴近现场、脱敏可复现的最小样例，呈现一段完整的价值落差：脏输入 → AI 处理前的问题 → 结构化交付物 → 审核通过 → 可复用清单。

## 六要素（按价值落差顺序逐项填）

1. **原始脏输入（脱敏）**
   > 落差的起点：贴近现场、被脱敏改写过的原始材料——缺字段的工单、口语化的 Key User 转述、夹中英文的描述、埋在截图里的报错等。

2. **AI 处理前的问题（缺什么、乱在哪）**
   > 这条输入到底差在哪：缺复现步骤？没有影响范围？分类错误？同一问题重复提交？把落差的"低点"摆清楚，后面的"高点"才有对比。

3. **结构化后的交付物（这一篇的主角）**
   > AI 整理后产出、可直接进 ITSM / 可被审核的结构化结果：带责任模块、影响范围、缺口标记的工单 / RCA 草稿 / 测试证据包 / 月报洞察初稿。这是证据场的主角——先让人看见"AI 把脏输入做成了什么"。

4. **审核通过后的定稿（人把关，然后过关）**
   > 顾问 / PMO / AMS Manager 把哪几处关：根因是否成立？影响范围是否漏？分类与优先级是否合理？——重点不在"有人审"，而在**审完通过、成为可交付的定稿**。让读者看到交付物真的过了这道闸，而不是停在草稿。

5. **AI 不能越权的边界（佐料，不是收口）**
   > 顺手交代 AI 在这一篇明确不做什么：不自动判定根因、不自动改配置、不自动上线、不自动审批、不替业务验收。这条是可信度的锚——点到为止即可，别让边界抢了交付物的戏，更不要把它当成证据场的落点。

6. **可复用的最小模板 / 检查清单（方法论资产 · 收口落点）**
   > 读者能直接拿走复用的一小段结构 / 清单：不是用来换关注的引流物料，而是可复现示例。证据场收在这里——让"做过、且做成了"的作品当最后一个画面。

## 脱敏与质感要求（硬约束）

- **不使用**任何客户真实原文、真实截图、真实日志、真实数据。
- 样例可来自真实交付经验的**抽象重构**，也可以是虚构案例，但必须经过脱敏、改写、场景重构，**不能反推出任何客户、系统、人员、业务或项目**。
- 但必须**高度贴近 SAP 乙方真实现场**——真实的字段、真实的痛点形态、真实的审核角色。证据的说服力来自现场质感与方法论可复现性，不来自数据是否为真客户。
- 措辞口径：用"脱敏样例 / 最小样例 / 交付物样例 / 检查清单 / 方法论资产 / 可复现示例"；不用"模板库 / 资料包 / 大礼包 / 领取 / 资源下载"。

## 一句话自检

> 把这个样例给一个资深 AMS Manager 看，他先看到的是"这人做成了一件拿得出手的东西"，还是只看到"这人很懂边界"？前者才算过关——交付物是主角，边界只是调味。

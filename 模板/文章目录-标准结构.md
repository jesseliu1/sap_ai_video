# 文章目录·标准结构

> 来自栏目目录约定。**不提前创建空文章目录**——只在文章实际开始、或已有内容需要归档时，才建这套目录。

## 每篇文章一个独立目录

```
<篇号>-<短标题>/
├── markdown/   文章稿（正文 .md）
├── html/       发布网页版本
├── assets/     本篇用到的图片、图表成品（正文以 ../assets/ 引用）
└── source/     图表 HTML、草稿、过程素材
```

## 约定细则

- 目录名建议：`02-Incident入口` 这样的「篇号-短标题」，便于排序和检索。
- 正文放 `markdown/`；正文里图片用相对路径 `../assets/<图名>.png` 引用。
- 成品图进 `assets/`；图表工程文件、草图、中间素材进 `source/`。
- 发布用的 HTML 进 `html/`。
- 被替换掉的旧版正文、废弃草稿，移到工作区根目录 `old/` 归档，不留在 `markdown/` 里干扰。

## 新建一篇时的最小动作

1. 在所属栏目目录下，按上面结构建该篇目录（四个子目录可按需先建 `markdown/`、`assets/`）。
2. 把《选题规划》里这一篇的约束抄进正文顶部当写作提示（类型、是否带入口前提句、是否必配证据场、专属立场口径）。
3. 写作时对照《模板/<对应文体>-写作清单.md》逐条过。
4. 定稿前用《速查/措辞口径速查表.md》扫一遍红线。
5. 发布后回填工作区根目录《发布管理表.md》。

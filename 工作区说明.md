# 工作区说明 ·「SAP交付AI笔记」写作底座

这个文件夹，是「SAP交付AI笔记」从策略到成稿的工作底座。

它的逻辑很简单：上面有一份不轻易动的**母文档**，定下账号的世界观、口径和边界；中间是一个个**栏目**，承载一季季的连续内容；每个栏目下面，是一篇篇按统一目录归档的**文章**。写作时左手翻**速查口径**、右手照**文体模板**，发完回填一张**发布管理表**——旧稿和被替换的版本，统一退进 `old/`。

母文档是地基，不常改；真正高频翻动的，是速查表和模板。先看口径，再动笔，定稿前再扫一遍红线，基本就不会写翻。

## 一篇文章从 0 到发布，大致这样走

1. 先去《栏目/…/选题规划-V2.0.md》认领下一篇，记下它的类型和专属约束（是否带入口前提句、是否必配证据场、有没有立场口径）。
2. 按《模板/文章目录-标准结构.md》给这一篇建目录——**不提前建空目录，写到哪篇才建哪篇**。
3. 对照对应的文体清单（场景 / 治理 / 经营）逐条写。
4. 尽量配一个脱敏样例（证据场六要素）。
5. 定稿前用《速查/措辞口径速查表.md》扫红线。
6. 图进 `assets/`、工程文件进 `source/`、发布版进 `html/`。
7. 回填《发布管理表.md》，旧稿退进 `old/`。

## 几条要记住的底线

- **价值在前，边界当调味**——先讲"我能带来什么、做过什么"，边界只在证明方案稳、不坑客户时一句带过。
- **入口质量决定 AI 上限**——先讲入口，再讲依赖入口的 RCA / 月报 / 知识沉淀。
- **第一层是策略不是天花板**——从 L1 切入，但要让人看见你看得见 L2/L3 的路。
- **经营账可以算，焦虑营销不能做**——人天定价在死、价值定价是机会，这是最强雇主钩子。
- **成功 = 能雇你的人读完更信你**——靠微信温好友分发，不追爆款 / 完读率 / 搜一搜。
